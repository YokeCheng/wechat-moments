## 2. 产品概述

### 2.1 产品定位

爆了么是一款面向内容创作者的一站式内容生产平台，服务对象包括个人创作者、MCN 机构、品牌运营人员。平台希望把内容生产的关键环节串成统一工作流：

`爆款发现 -> 提示词选择/打造 -> AI 生文 -> 一键排版 -> 公众号管理/发布`

### 2.2 产品价值

- 降低选题成本：快速发现爆款文章和热搜话题。
- 提升生产效率：通过提示词和 AI 生成减少写作时间。
- 保持输出质量：通过模板、分组、排版预览统一产出标准。
- 缩短发布链路：从选题到发布尽量在一个平台内完成。

### 2.3 当前阶段判断

当前前端已形成完整信息架构和高保真交互原型，但除联系表单外，大部分能力仍处于本地 mock 阶段。后端第一阶段应优先打通以下最小闭环：

1. 爆款列表查询
2. 提示词库管理
3. AI 生文任务与文章草稿管理
4. 排版草稿保存与 HTML 导出

公众号授权发布、支付、会员权益作为第二阶段能力接入。

## 3. 项目背景

内容创作者在实际工作中通常面临以下问题：

- 热点分散在多个平台，选题效率低。
- AI 提示词管理混乱，复用成本高。
- 生文、排版、发布分散在多个工具中，流程割裂。
- 团队缺少统一内容资产沉淀和复盘入口。

因此需要一个围绕内容生产主链路设计的统一平台，先解决创作效率问题，再逐步扩展到渠道管理和商业化。

## 4. 目标与非目标

### 4.1 目标

#### 业务目标

- 让用户能在一个系统内完成从选题到排版的主要动作。
- 建立内容资产沉淀能力，包括提示词、文章草稿、排版稿。
- 为后续会员订阅、公众号发布和团队协作预留扩展空间。

#### 产品目标

- 打通 P0 主链路：爆款发现 -> 提示词 -> 生文 -> 排版。
- 保证所有关键页面都有对应后端数据源，不再依赖前端本地 mock。
- 设计统一的数据模型和任务状态流，避免后续返工。

#### 体验目标

- 页面首屏响应快，列表筛选稳定。
- AI 生成和发布动作均有明确状态反馈。
- 草稿和历史记录可追踪、可恢复。

### 4.2 非目标

以下内容不作为 v0.1 后端必须交付项：

- 完整团队协作与多角色审批
- 全量第三方平台自动采集
- 复杂 BI 报表
- 实时聊天客服系统
- 多语言国际化落地

## 5. 用户角色

### 5.1 个人创作者

- 关注点：找热点、快速生成文章、快速排版
- 核心诉求：省时间、操作简单、成本可控

### 5.2 MCN / 内容运营

- 关注点：多账号管理、内容资产沉淀、批量生产
- 核心诉求：统一管理、流程标准化、可复用

### 5.3 品牌运营

- 关注点：行业热点、风格稳定、可控发布
- 核心诉求：内容一致性、节奏稳定、数据可追溯

## 6. 使用场景

### 场景 1：热点仿写

1. 用户进入每日爆款页，筛选公众号或头条号热门文章
2. 选择一篇文章，点击“提示词”或“仿写”
3. 系统带入标题、参考链接、提示词
4. 用户在智能生文页发起生成
5. 生成完成后进入排版页美化内容
6. 用户复制 HTML，后续手动发布或接入公众号发布

### 场景 2：自定义选题创作

1. 用户在提示词库选择合适模板
2. 进入生文页输入主题、分组、参考素材
3. 系统生成文章草稿
4. 用户编辑后进入排版页
5. 保存排版草稿，后续继续编辑或发布

### 场景 3：长期内容资产管理

1. 用户沉淀提示词分类和提示词模板
2. 文章按分组管理，形成可复用选题库
3. 排版稿和发布记录可追溯

## 8. 功能需求

### 8.1 每日爆款

#### 功能描述

聚合热门内容与热搜话题，帮助用户发现创作方向。

#### 子模块

- 公众号爆款
- 头条号爆款
- 今日热搜榜

#### 需求点

- 支持按关键词搜索文章标题和作者。
- 支持按领域分类筛选。
- 支持按时间范围筛选。
- 支持按阅读量区间筛选。
- 支持查看文章基础指标：阅读量、点赞数、分享数。
- 支持收藏文章。
- 支持从文章一键生成提示词。
- 支持从文章一键仿写并带入生文页。
- 支持导出列表结果。

#### P0 验收标准

- 用户能按平台、关键词、分类、阅读量获取文章列表。
- 用户能点击“仿写”进入生文页并带入必要参数。
- 用户能点击“提示词”触发提示词生成能力。

### 8.2 提示词库

#### 功能描述

沉淀和管理用于 AI 创作的提示词资产。

#### 需求点

- 支持提示词分类列表展示。
- 支持新增、编辑、删除分类。
- 支持提示词列表按分类和关键词筛选。
- 支持新增、编辑、删除提示词。
- 支持提示词状态管理：草稿、启用中、生成中。
- 支持查看提示词详情。
- 支持复制提示词文本。
- 支持记录提示词使用次数。
- 支持从提示词直接进入智能生文。

#### P0 验收标准

- 用户能完成提示词分类和提示词的基本 CRUD。
- 用户能查看启用状态并切换状态。
- 用户能从提示词详情页进入生文流程。

### 8.3 智能生文

#### 功能描述

围绕文章生成任务管理生成内容草稿，并支持后续编辑与排版。

#### 需求点

- 支持文章分组管理。
- 支持文章列表展示：标题、分组、提示词、状态、字数、创建时间。
- 支持新建生成任务。
- 生成参数包括：标题、分组、提示词、参考链接、图片数量。
- 支持文章状态流转：草稿、生成中、已完成、失败。
- 支持编辑文章元信息。
- 支持删除文章。
- 支持将已完成文章发送到排版页。
- 支持承接来自爆款页和提示词库的预填参数。

#### 状态定义

- `draft`：草稿，尚未生成或生成后待编辑
- `generating`：生成中
- `completed`：已生成完成
- `failed`：生成失败

#### P0 验收标准

- 用户能发起生成任务并看到状态变化。
- 用户能查询历史文章并编辑元信息。
- 用户能把已完成文章带入排版页。

### 8.4 一键排版

#### 功能描述

将文章 Markdown 内容转换为适合公众号发布的 HTML，并提供实时预览。

#### 需求点

- 支持标题与封面设置。
- 支持 Markdown 编辑。
- 支持工具栏快捷插入标题、列表、引用、链接、图片、代码块等。
- 支持主题样式、字体、字号、行高、标题对齐、段落缩进等设置。
- 支持手机/平板预览。
- 支持全屏预览。
- 支持复制排版结果 HTML。
- 支持保存排版草稿。
- 支持继续编辑历史排版稿。

#### P0 验收标准

- 用户能从生文页带入内容进入排版页。
- 用户能保存排版草稿。
- 用户能复制 HTML 并保留基本样式。

### 8.5 公众号管理

#### 功能描述

管理用户已授权的公众号账号，为后续发布提供基础能力。

#### 需求点

- 支持公众号授权入口。
- 支持已授权账号列表展示。
- 支持查看账号基础数据。
- 支持发起文章发布。
- 支持记录发布结果。
- 支持解绑账号。

#### P1 验收标准

- 用户能看到自己的公众号列表。
- 用户能完成授权回调后的账号入库。
- 用户能对指定文章发起发布。

### 8.6 会员中心

#### 功能描述

通过套餐与权益控制平台能力，支撑商业化。

#### 需求点

- 支持套餐展示。
- 支持当前订阅状态展示。
- 支持下单和支付状态回调。
- 支持资源额度控制：爆款查看量、提示词数、生文次数、导出额度、授权公众号数量。
- 支持查询使用量。

#### P1 验收标准

- 用户能查看套餐与当前权益。
- 订单支付成功后用户权益能够生效。

### 8.7 联系我们

#### 功能描述

收集用户反馈、合作需求与账号问题。

#### 需求点

- 支持联系方式展示。
- 支持提交留言表单。
- 支持反馈类型区分。
- 支持后台查询留言记录。

#### P1 验收标准

- 用户能提交留言。
- 平台能在后台查看留言信息。

## 10. 核心业务规则

- 文章、提示词、分组、排版稿默认按用户隔离。
- 删除分类时，需校验分类下是否仍有关联提示词。
- 删除分组时，不强删文章，文章可回落到“未分组”。
- 文章生成属于异步任务，必须有任务状态和错误信息。
- 只有已完成的文章才允许进入排版发布流程。
- 订阅权益变更后，资源额度应立即刷新。
- 联系表单要限制消息长度和必填字段。

## 11. 非功能需求

### 11.1 性能

- 列表接口默认响应时间目标小于 500ms。
- 分页查询需支持至少万级数据。

### 11.2 稳定性

- 异步任务失败不能影响已存在文章数据。
- 发布任务要有重试与失败记录。

### 11.3 安全

- 用户数据需要按账号隔离。
- 重要写接口需要鉴权。
- 发布回调和支付回调需要验签。

### 11.4 可扩展性

- 接口与数据表需支持新增平台类型。
- 任务系统需支持后续接入队列或 worker。

## 12. 核心指标

- 每日活跃创作者数
- 生文任务发起数
- 生文完成率
- 提示词使用率
- 排版稿保存率
- 会员转化率
- 联系表单提交量

## 13. 里程碑建议

### 阶段 1：内容生产主链路

- 完成爆款、提示词、生文、排版 P0 接口
- 用真实数据库替换前端 mock
- 同步建设最小 Harness 执行层：
  - 核心 Harness 文档存在性检查
  - `backend_spec.yaml` OpenAPI 解析检查
  - Slice / route / mock 关键漂移扫描

### 阶段 2：发布与商业化

- 接入公众号管理
- 接入会员、订单、额度控制

### 阶段 3：运营与协作

- 教程后台化
- 团队协作与审批
- 数据分析与报表

## 14. 风险与待确认事项

- 爆款文章与热搜数据的真实来源尚未确定，需明确抓取或第三方 API 方案。
- AI 生文服务依赖的模型、调用成本、并发策略尚未确定。
- 公众号授权与发布需明确微信开放平台接入方案。
- 支付方式、支付服务商和票据流程尚未确定。
- 当前前端部分文件存在编码不一致和未接通路由问题，后端对接前需同步清理。
